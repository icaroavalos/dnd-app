# Generated Artifacts Policy

## Directory Status

| Directory | Content | Git Status |
|-----------|---------|------------|
| `dist/` | Frontend TypeScript build output | IGNORED |
| `backend/dist/` | Backend NestJS build output | IGNORED |
| `backend/generated/` | Prisma client generated code | IGNORED |

## Policy

**Generated artifacts MUST NOT be committed to Git.**

These directories are produced by build tools and should be regenerated locally:

- `dist/` - Generated by `npm run build` (TypeScript compilation)
- `backend/dist/` - Generated by `npm --prefix backend run build` (NestJS compilation)
- `backend/generated/` - Generated by Prisma during `npm install` or `prisma generate`

## Why Not Commit Generated Code

1. **Source of truth**: TypeScript sources (`.ts`) are the canonical code
2. **Build artifacts vary**: Different Node versions or TypeScript flags produce different output
3. **Noise in reviews**: Build output changes obscure actual code changes
4. **Repository bloat**: Generated files can be 10x larger than sources

## If Build Fails

Run the appropriate build command:

```bash
# Frontend
npm run build

# Backend
npm --prefix backend run build

# Both
npm run build && npm --prefix backend run build
```

## Git Configuration

The `.gitignore` file already contains:

```
dist/
backend/dist/
backend/generated/
```

If you see generated files in `git status`, they were committed before this policy.
Remove them with:

```bash
git rm --cached -r dist/ backend/dist/
git commit -m "chore: remove generated build artifacts"
```
